Afbeelding 2 Resultaat Felix

Individueel heb ik al best wat leerdoelen aangetoond met het maken van mijn chat applicatie Felix. In mijn leerplan had ik al wat leerdoelen uitgekozen die ik in mijn chat applicatie wilde aantonen. De leerdoelen die ik gekozen had zijn als volgt. De leerdoelen die doorstreept zijn heb ik aangetoond in mijn chat applicatie.

  • Google authenticator two-factor authentication;
  • Make a misuse case (or abuse case) design;
  • Fix some security bugs in an application or system environment;
  • Implement data encryption on application code level;
  • Add security logging to your design, implementation and testing;
  • Add fuzzing tests for an application that you are developing;
  • Join the Red Team - Blue Team test event;
  • Implement the AES algorithm in your favorite programming language, or, Implement the RSA algorithm in your favorite programming language;
  • Is your implemented (AES or RSA software) protected against side channel attacks.

Hieruit blijkt dat ik nog de fuzzing tests wil aantonen. Deze wil ik aantonen in de volgende helft van dit semester, fuzzing tests kan ook alleen effectief uitgevoerd worden op een werkende applicatie dus het is logisch dat deze in een later stadium uitgevoerd wordt.

De leerdoelen voor de losse opdrachten stonden al gepland voor de tweede helft. Volgens mijn planning die ook in mijn leerdoel pagina staan, loop ik precies op schema.
